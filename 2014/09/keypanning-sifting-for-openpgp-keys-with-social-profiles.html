Keypanning: sifting for OpenPGP keys with social profiles

How do you find the PGP key for someone who you only know via a Twitter, G+ or Github profile? Add your profile url as a user id on your key, and publish a fingerprint on your profile site.

For example, I've published mine on Google+, on Github and on Twitter.

Clients can now find your keys in the distributed keyservers (eg: this query finds keys that match my github profile) and select ones that have a matching fingerprint on profile sites linked from the key.

This provides a comparable degree of assurance a service like keybase offers -- namely, a person who controls the key was able to publish to that profile site. But it doesn't need the complication of an additional service, as clients can directly check the profile sites after obtaining candidate keys.

I've implemented a sample client which does just this. It has a CLI as well as a nicer-looking web interface from a locally runnable web server.

Given a profile URL, it searches and then automatically looks for matching fingerprints in profiles linked from the key. The github page has more details as well as instructions for publishing your keys with gpg using this technique.
